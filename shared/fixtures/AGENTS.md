# Shared Fixtures

## Owner
- Agent 1 owns this directory.

## Goal
- Provide stable, realistic payloads so the frontend and API can progress in parallel.

## Fixture Expectations
- Derive fixtures from the real challenge data, not invented toy examples.
- Include at least:
  - overview payload
  - one process detail payload
  - one opportunity detail payload
  - one workflow-generation input and output sample

## Rules
- Keep fixtures small enough for fast iteration.
- Use deterministic file names.
- Note the source process family and time window in a short fixture README if useful.
